## Directory Purposes

**`app/`:**
- Purpose: Next.js 13 app directory containing pages, layouts, and API routes
- Contains: Page components (TSX), layouts, global styles, API route handlers
- Key files: `page.tsx` (main app), `layout.tsx` (root layout)

**`components/`:**
- Purpose: Reusable React components organized by feature domain
- Contains: Feature-based components (auth, goals, layout, contact, setup) plus UI primitive components
- Key files: `goal-card.tsx` (displays single goal), `goal-form.tsx` (create/edit), `hundee-wall.tsx` (public feed)

**`components/ui/`:**
- Purpose: Shadcn UI component library
- Contains: Unstyled, accessible primitives (Button, Card, Dialog, Form, etc.)
- Generated: Yes, via Shadcn CLI
- Committed: Yes
- Pattern: All use Radix UI for accessibility, styled with Tailwind

**`hooks/`:**
- Purpose: Custom React hooks for shared state logic
- Contains: `useAuth()` for authentication state, `useToast()` wrapper for notifications
- Key pattern: `useAuth()` runs on component mount, subscribes to Supabase auth changes

**`lib/`:**
- Purpose: Business logic, utilities, and Supabase operations
- Contains: Async functions for goals, auth, setup verification, and helper utilities
- Key files: `supabase.ts` (client + types), `goals.ts` (CRUD), `auth.ts` (auth functions)

**`supabase/migrations/`:**
- Purpose: Database schema versioning via SQL migrations
- Contains: SQL files with schema definitions and RLS policies
- Pattern: Filename format: `YYYYMMDDHHMMSS_description.sql`
- Key tables: profiles, goals, goal_updates

**`types/`:**
- Purpose: Custom TypeScript type definitions
- Contains: Currently only `react-hcaptcha.d.ts` (hCaptcha library types)

## Naming Conventions

**Files:**
- Components: kebab-case with .tsx extension (e.g., `auth-form.tsx`, `goal-card.tsx`)
- Utilities/Hooks: kebab-case with .ts extension (e.g., `use-auth.ts`, `verify-setup.ts`)
- Pages: kebab-case with .tsx extension inside folders (e.g., `app/setup/page.tsx`)
- Migrations: YYYYMMDDHHMMSS_snake_case.sql

**Directories:**
- Lowercase, single words or hyphenated (e.g., `components`, `lib`, `supabase`, `goal-updates`)
- Feature-based grouping inside components (e.g., `components/goals/`, `components/auth/`)

**Functions:**
- camelCase for all functions and exported utilities (e.g., `createGoal()`, `getUserGoals()`, `updateGoalProgress()`)
- Async functions are not prefixed but marked with `async` keyword
- React hooks prefixed with `use` (e.g., `useAuth()`)

**Variables:**
- camelCase for local and state variables (e.g., `goals`, `loading`, `editingGoal`)
- UPPERCASE for constants (e.g., `EMOJI_OPTIONS`, `COLOR_OPTIONS`, `RATE_LIMIT_WINDOW_MS`)

**Types:**
- PascalCase for all type definitions (e.g., `Goal`, `Profile`, `GoalUpdate`, `GoalFormProps`)
- Interface suffixes use Props for component props (e.g., `GoalFormProps`, `GoalCardProps`)

## Where to Add New Code

**New Feature:**
- Primary code: `components/[feature-name]/` for UI, `lib/[feature-name].ts` for logic
- Tests: Not yet implemented; would go in `__tests__/` directories
- Example: New "categories" feature would have `components/categories/category-card.tsx` and `lib/categories.ts`

**New Component/Module:**
- Implementation: Place in `components/` directory, grouped by feature
- Shadcn UI components: Regenerate with `npx shadcn-ui@latest add [component]` to `components/ui/`
- Hooks: Create in `hooks/` with `use` prefix
- API routes: Create in `app/api/[route-name]/route.ts`

**Utilities:**
- Shared helpers: `lib/utils.ts` (currently has `cn()` for classNames merge)
- Supabase operations: `lib/supabase.ts` (client only) or `lib/[entity].ts` (operations)
- Setup/verification: `lib/verify-setup.ts`

## Special Directories

**`app/api/`:**
- Purpose: Next.js API routes (server-side only)
- Generated: No
- Committed: Yes
- Pattern: One folder per endpoint, `route.ts` file handles POST/GET/etc.
- Key feature: Service role key available, rate limiting, CAPTCHA verification

**`components/ui/`:**
- Purpose: Shadcn UI components
- Generated: Yes, via `npx shadcn-ui@latest add [component]`
- Committed: Yes (components are source code, not dependencies)
- Pattern: Minimal unstyled, uses Radix UI + Tailwind, exports named exports

**`.next/`:**
- Purpose: Next.js build output
- Generated: Yes, by `next build`
- Committed: No (gitignored)

**`public/`:**
- Purpose: Static assets (images, fonts, etc.)
- Generated: Partially (Next.js font files)
- Committed: User-added static files only

**`node_modules/`:**
- Purpose: Package dependencies
- Generated: Yes, by npm install
- Committed: No (gitignored)
